# Contributing to MQGT-SCF

Thank you for your interest in contributing! This document provides guidelines for contributing to the empirical validation of the Theory of Everything.

## Code of Conduct

- Be respectful and inclusive
- Focus on scientific rigor
- Maintain reproducibility
- Document all changes

## How to Contribute

### Reporting Issues

- Use GitHub Issues
- Include reproducible examples
- Provide error messages and logs
- Reference relevant documentation

### Submitting Changes

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Follow code style guidelines
   - Add tests if applicable
   - Update documentation
4. **Commit your changes**
   ```bash
   git commit -m "Add: Description of changes"
   ```
5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Create a Pull Request**

## Code Style

### Python

- Follow PEP 8
- Use type hints where possible
- Document functions with docstrings
- Maximum line length: 100 characters

### Documentation

- Use Markdown for documentation
- Include code examples
- Explain scientific rationale
- Cite sources

## Testing

- Run existing tests before submitting
- Add tests for new features
- Ensure reproducibility
- Test across platforms if possible

## Scientific Contributions

### Experimental Data

- Include provenance
- Provide SHA256 hashes
- Document digitization methods
- Reference original publications

### Analysis

- Document methodology
- Include statistical analysis
- Acknowledge limitations
- Suggest future work
