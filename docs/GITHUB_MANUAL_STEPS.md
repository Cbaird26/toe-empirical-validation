# GitHub Manual Steps (Phase B)

These steps must be done once on GitHub by a repository owner. They complete the verification checklist and improve discoverability.

**Repository:** https://github.com/Cbaird26/toe-empirical-validation

---

## 1. Enable GitHub Pages

1. Go to: https://github.com/Cbaird26/toe-empirical-validation/settings/pages  
2. Under **Source:** select **GitHub Actions**  
3. Click **Save**  
4. After the first workflow run (~2 minutes), the site will be live at:  
   **https://cbaird26.github.io/toe-empirical-validation/**

---

## 2. Add Repository Topics

1. Go to: https://github.com/Cbaird26/toe-empirical-validation  
2. Click the gear icon next to **About**  
3. In **Topics**, add:
   - `theory-of-everything`
   - `physics`
   - `empirical-validation`
   - `quantum-gravity`
   - `consciousness`
   - `open-science`
   - `mqgt-scf`
   - `scientific-computing`
4. Click **Save changes**

---

## 3. Create First Release

1. Go to: https://github.com/Cbaird26/toe-empirical-validation/releases/new  
2. **Tag:** `v1.0.0` (create new tag)  
3. **Release title:** `Initial Release: Empirical Validation of MQGT-SCF`  
4. **Description:** Use or adapt the text in [PUBLICATION_README.md](../PUBLICATION_README.md)  
5. Optionally attach **A Theory of Everything - ZoraASI - C.M., Baird (2026).pdf** (or the compiled `paper/main.pdf`) as a release asset  
6. Click **Publish release**
